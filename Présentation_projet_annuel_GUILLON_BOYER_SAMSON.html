PARTIE 1 – Site Web d’Aide à l’Alternance

1. Objectifs et public cible

Le site a été conçu pour répondre à un besoin récurrent chez les étudiants en informatique :

trouver une alternance dans le secteur IT de manière efficace et personnalisée. L’objectif principal est d’aider les alternants à :

  • Créer un CV pertinent à l’aide d’un générateur automatisé
  • Accéder à des offres ciblées et actualisées
  • Postuler en quelques clics
  • Suivre leur parcours et leurs compétences
  • Profiter de ressources utiles pour préparer leur projet professionnel

Mais le site s’adresse également aux recruteurs. Ces derniers peuvent :

  • Déposer facilement des offres avec critères et compétences précises
  • Accéder à une base de profils d’alternants engagés et en cours de formation
  • Visualiser des CV formatés automatiquement selon les données réelles du profil
  • Communiquer avec les candidats via une messagerie intégrée

Public cible :

  • Étudiants en informatique, en quête d’un contrat d’alternance, souvent en reconversion, en début de parcours ou avec peu de réseau professionnel
  • Recruteurs, RH et entreprises du secteur IT souhaitant trouver rapidement des profils motivés et adaptés à leurs besoins techniques

2. Fonctionnalités principales

Le site s’articule autour de deux espaces : Candidat et Recruteur, accessibles après authentification. Voici les principales fonctionnalités offertes :

Espace Candidat :

  • Création de profil complet (ville, école, niveau, objectifs, lien LinkedIn/portfolio)
  • Génération de CV personnalisé au format PDF à partir des données saisies
  • Consultation et recherche d’annonces d’alternance
  • Postulation directe aux offres
  • Suivi des candidatures envoyées
  • Messagerie interne avec les recruteurs
  • Suivi et évaluation des compétences via une interface dédiée
  • Ajout d’expériences professionnelles pour enrichir le CV

Espace Recruteur :

  • Dépôt d’annonces avec description, titre et compétences recherchées
  • Consultation des profils et des CV des candidats
  • Gestion des candidatures reçues
  • Messagerie pour échanger avec les candidats

Espace Administrateur :

  • Gestion des utilisateurs (création, modification, suppression)
  • Modération des annonces postées
  • Supervision de l'activité générale (candidatures, échanges, publications)
  • Accès aux données statistiques d’usage de la plateforme
  • Maintenance des compétences proposées pour les annonces et les profils

Autres fonctionnalités :

  • Page d’accueil avec présentation du projet
  • Page de contact
  • Accès à des ressources : conseils CV, préparation à l’entretien, suivi de progression

3. Interface utilisateur

Le design a été pensé pour être simple, moderne, lisible et responsive.

  • Utilisation de la police Poppins pour une lecture fluide
  • Palette de couleurs centrée autour de tons verts et bleus (apaisants et professionnels)
  • Sidebar latérale pour la navigation (fixe sur desktop, adaptative sur mobile)
  • Zones principales : contenu centré, encadré, avec des boîtes thématiques (profil, projets, formulaire...)
  • Interface formulaire ergonomique et accessible
  • Mise en avant des boutons d'action (postuler, contacter, télécharger CV...)

L’interface s’adapte aux différents formats d’écrans, rendant l’outil utilisable aussi bien sur desktop que mobile.

4. Structure technique du site

Le site est structuré autour de fichiers PHP modulaires qui gèrent les différentes pages et logiques du site :

  • index.php : page d’accueil
  • login.php, logout.php, register.php : gestion des sessions utilisateurs
  • profil.php, dashboard.php : espace personnel candidat
  • creer_cv.php, generate_cv.php, cv.php : gestion du CV et génération PDF via TCPDF
  • offres.php, annonces.php, postuler.php : affichage des offres et interaction avec les recruteurs
  • candidatures.php : suivi des candidatures
  • competences.php, suivi_competences.php : interface de suivi des compétences
  • messages.php : messagerie interne
  • ressources.php, contact.php : ressources et communication

Le style est géré par un fichier CSS principal (style.css) avec des classes spécifiques pour chaque zone fonctionnelle.

L’ensemble du site repose sur une base de données MySQL nommée alternance_it, interfacée via PHP natif sans framework, pour une meilleure maîtrise pédagogique du code.

PARTIE 2 – Serveur Web embarqué ESP32

1. Objectif technique

Créer un serveur web miniature autonome fonctionnant sur un microcontrôleur ESP32 WROVER, capable de :

  • Fournir une interface web accessible localement (sans Internet)
  • Gérer l’hébergement de pages HTML/CSS/JS
  • Permettre l’envoi, la suppression et la gestion de fichiers via une carte SD
  • Protéger l’accès grâce à un système d’authentification sécurisé basé sur des cookies et un hash SHA-256
  • Communiquer via une API REST locale

Ce serveur est conçu pour un usage personnel, pédagogique ou de prototypage, sans dépendre d’un hébergement externe.

2. Composants matériels

  • ESP32 WROVER : microcontrôleur WiFi/Bluetooth avec mémoire étendue
  • Carte microSD : stockage des fichiers hébergés, format FAT32 jusqu’à 32 Go
  • Alimentation 5V 1A : alimentation standard micro-USB
  • Accès WiFi local : via point d’accès ESP32 ou réseau domestique

3. Architecture logicielle (Arduino)

Le projet est découpé en fichiers .ino spécialisés pour garder une structure modulaire et maintenable :

  • web-server.ino : point d’entrée du projet, initialise WiFi, carte SD, base utilisateurs, démarre le serveur
  • server.ino : déclaration du serveur HTTP via ESPAsyncWebServer
  • router.ino : définition des routes HTTP (/, /api, /upload, etc.)
  • api.ino : gestion de l’authentification (login, logout, vérification de session), API REST en JSON
  • upload.ino : gestion de l’envoi de fichiers depuis un formulaire web
  • kvdb.ino : base de données clé/valeur rudimentaire pour stocker les utilisateurs
  • internal.ino, utils.ino : fonctions utilitaires (conversion taille, format fichiers, gestion des chemins...)

4. Fonctionnalités du serveur

  • Hébergement de fichiers HTML/CSS/JS stockés sur carte SD
  • Explorateur de fichiers web : interface avec options de téléversement, renommage, suppression
  • Authentification sécurisée : mot de passe stocké en SHA-256 + gestion de session via cookies HTTP
  • API REST : envoie et réception de requêtes JSON pour interagir avec les fichiers ou sessions
  • Interface admin avec navigation à onglets, mise à jour asynchrone

5. Problèmes rencontrés et solutions

  • HTTPS instable : limitations de l’ESP32 sur le SSL → usage local recommandé
  • Cookies parfois bloqués par Firefox : ajout d’en-têtes supplémentaires pour contourner les restrictions
  • Gestion multipart complexe : solution personnalisée pour découper et réassembler les fichiers uploadés
  • Limitations dynamiques de l’ESP32 : usage optimisé du buffer et chargement progressif

6. Résultat obtenu

  • Serveur local autonome, entièrement fonctionnel et rapide
  • Authentification robuste et session utilisateur active
  • API REST testée sur navigateur (JSON preview)
  • Interface web fluide, ergonomique et compatible mobile

PARTIE 3 – Base de Données alternance_it

1. Objectifs de la base

La base de données MySQL alternance_it permet de gérer toutes les interactions entre les utilisateurs, les offres, les compétences et les candidatures. Elle soutient à la fois les espaces candidat et recruteur, tout en permettant la génération dynamique de CV et le suivi des parcours.

2. Schéma relationnel (structure)

Tables principales :

  • utilisateurs : contient les données de connexion et le rôle (candidat, recruteur, admin)
  • profil_utilisateur : informations détaillées (ville, école, objectifs, liens...)
  • annonces : offres d’alternance publiées par les recruteurs
  • candidatures : lien entre utilisateurs et annonces (avec statut de suivi)
  • competences : compétences disponibles dans la plateforme (catégories, libellé)
  • utilisateurs_competences : association entre utilisateur et compétences avec niveau
  • annonces_competences : compétences requises pour chaque offre
  • experiences_professionnelles : expériences renseignées pour enrichir le CV
  • messages : échanges internes entre utilisateurs

3. Logique métier et interactions

La base est conçue pour permettre une logique métier riche :

  • Requêtes de type "matching" : croisement entre compétences d’un utilisateur et exigences des annonces
  • Génération dynamique du CV à partir de : profil, expériences, compétences sélectionnées
  • Filtrage intelligent des offres selon le profil
  • Suivi du niveau d’acquisition des compétences avec évaluation
  • Possibilité pour les recruteurs de trier les candidats selon les compétences clés

4. Requêtes SQL utiles (exemples)

  • Afficher les offres compatibles avec les compétences d’un candidat :
SELECT a.* FROM annonces a
JOIN annonces_competences ac ON a.id = ac.annonce_id
JOIN utilisateurs_competences uc ON ac.competence_id = uc.competence_id
WHERE uc.utilisateur_id = :id_utilisateur
GROUP BY a.id;
  • Lister les candidatures envoyées par un utilisateur :
SELECT a.titre, c.date_postulation, c.statut
FROM candidatures c
JOIN annonces a ON c.annonce_id = a.id
WHERE c.utilisateur_id = :id_utilisateur;
  • Afficher le profil complet d’un utilisateur pour le CV :
SELECT u.nom, u.email, p.ville, p.ecole, e.intitule, e.description, c.libelle
FROM utilisateurs u
JOIN profil_utilisateur p ON u.id = p.utilisateur_id
LEFT JOIN experiences_professionnelles e ON u.id = e.utilisateur_id
LEFT JOIN utilisateurs_competences uc ON u.id = uc.utilisateur_id
LEFT JOIN competences c ON uc.competence_id = c.id
WHERE u.id = :id_utilisateur;

5. Résumé

La base alternance_it constitue la colonne vertébrale du projet. Elle permet non seulement de stocker les données des utilisateurs, mais aussi d’offrir une logique d’interaction riche et évolutive : adaptation des offres, génération de CV, suivi des progrès. Sa structure relationnelle rend le système robuste, évolutif, et prêt à être intégré dans une application à plus grande échelle.

PARTIE 4 – Synthèse, réflexion et mise en perspective

1. Lien entre le site, le serveur embarqué et la base de données

Le projet dans sa globalité a été pensé comme une solution cohérente, modulaire et interconnectée, répondant à une problématique concrète : accompagner les étudiants en informatique dans leur recherche d’alternance.

  • Le site web est la façade visible de la plateforme, proposant aux utilisateurs une interface complète pour consulter, postuler et suivre des offres d’alternance, tout en construisant leur identité professionnelle.
  • La base de données alternance_it est le socle qui rend possible l’intelligence fonctionnelle du site : elle permet la personnalisation, le suivi des candidatures, la correspondance profil/offre, et la gestion du parcours utilisateur.
  • Le serveur ESP32 incarne quant à lui une approche complémentaire : il démontre la possibilité de proposer un hébergement local autonome et sécurisé, utile en cas de déploiement sur des réseaux fermés, en atelier pédagogique, ou en environnement contraint (pas d’accès internet, faible consommation).

L’intégration de ces éléments permet d’imaginer des usages avancés :

  • Utilisation du serveur ESP32 comme outil de secours pédagogique pour les étudiants en cas de coupure réseau.
  • Déploiement local sur une borne WiFi autonome pour des salons ou ateliers de formation.
  • Sauvegarde temporaire ou transfert sécurisé de CV et profils via carte SD.

2. Fonctionnalités résolutives et contributions concrètes

Voici quelques exemples de problèmes concrets adressés par la solution :

  • Manque d’outils pour créer un CV pertinent sans compétences avancées en bureautique → Résolu via la génération automatique de CV PDF.
  • Difficulté à cibler des offres correspondant à son profil → Résolu par le filtrage intelligent basé sur les compétences stockées en base.
  • Suivi de progression personnel inexistant → Suivi des compétences et expériences enrichissantes via interface dédiée.
  • Risque de coupure réseau ou besoin de déploiement temporaire → Contourné grâce à l’ESP32 et au serveur web local embarqué.

Le projet apporte donc une solution pragmatique, hybride et pédagogique, s’appuyant sur des technologies open source et du matériel accessible.

3. Technologies utilisées

  • Frontend : HTML5, CSS3 (design responsive), JS natif
  • Backend Web : PHP natif, TCPDF pour la génération PDF
  • Base de données : MySQL
  • Matériel embarqué : ESP32 WROVER + carte microSD
  • Firmware ESP32 : Arduino (C++) + ESPAsyncWebServer + SPI + SHA-256

4. Problèmes rencontrés

  • Difficulté initiale à structurer la base de données pour anticiper les croisements métier
  • Incompatibilités de cookies sur certains navigateurs (Firefox) → nécessitant des tests et des contournements manuels
  • Complexité de l’upload multipart sur ESP32 → nécessité de découpage et recomposition fragmentaire
  • Débogage des échanges API REST avec ESP32
  • Gestion de sessions PHP et C++ de manière parallèle mais cohérente

5. Ce que ce projet nous a apporté

  • Vision complète d’un projet web professionnel, de la conception à la mise en ligne
  • Maîtrise des outils open source, des bases de données relationnelles et des serveurs web
  • Introduction à la programmation embarquée et à ses contraintes matérielles
  • Travail en équipe, avec répartition claire des rôles (frontend, backend, API, embarqué...)
  • Approche concrète des problématiques métiers rencontrées par les alternants et mise en œuvre de solutions adaptées

Ce projet représente un socle solide pour aborder la suite de notre parcours professionnel, alliant rigueur technique, utilité réelle, et transversalité des compétences.

PARTIE 5 – Conclusion générale

Ce projet annuel nous a permis de concevoir une solution numérique complète, ancrée dans une problématique réelle : l’accompagnement des étudiants et alternants dans la recherche d’une opportunité professionnelle dans le secteur IT.

En alliant un site web fonctionnel, une base de données structurée et un serveur embarqué ESP32 autonome, nous avons exploré différents aspects du développement informatique :

  • Développement web full stack (PHP, MySQL, CSS, HTML, TCPDF)
  • Modélisation et exploitation de base de données relationnelle
  • Programmation embarquée et communication réseau via API REST
  • Mise en place d’un système d’authentification sécurisé, compatible sur plusieurs environnements

Le projet nous a confrontés à des situations concrètes : limitations techniques, débogage d’erreurs complexes, réflexion sur l’ergonomie, sécurisation des échanges, et compatibilité inter-navigateurs. Ces défis ont été autant d’opportunités d’apprentissage technique que de développement de notre autonomie et de notre capacité à travailler en équipe.

Il nous a également permis de nous interroger sur les besoins réels des alternants et sur les outils qui pourraient améliorer leur insertion professionnelle.

Nous retenons de ce projet :

  • Une vision d’ensemble d’un écosystème applicatif, de la conception à la documentation
  • Une maîtrise approfondie des technologies modernes, tant web qu’embarquées
  • Une capacité à structurer notre travail et à coopérer efficacement

Ce projet a été une expérience riche, formatrice, et concrète, préparant chacun d’entre nous aux exigences du monde professionnel.

1. Présentation globale

Ce projet repose sur trois piliers interconnectés :

  • Un site web complet en PHP/MySQL permettant à des alternants de créer leur profil, générer un CV dynamique, postuler à des offres, suivre leurs compétences.
  • Une base de données relationnelle structurée et pensée pour l’adéquation profil/offre.
  • Un serveur embarqué ESP32 capable d’héberger une interface web locale, de gérer l’upload de fichiers via carte SD et de sécuriser l’accès avec une API REST.

Il vise à démontrer la faisabilité d’un environnement hybride, mêlant développement web classique, programmation embarquée et logique métier structurée.

I. Architecture du site web (PHP/MySQL)

1.1 Technologies utilisées

  • Frontend : HTML5, CSS3, JS vanilla (responsive design)
  • Backend : PHP 8+ (sans framework)
  • Génération PDF : Librairie TCPDF
  • Base de données : MySQL 5.7/8.0

1.2 Organisation des fichiers

  • index.php : page d’accueil
  • login.php, logout.php, register.php : authentification utilisateur avec gestion de session PHP native (via $_SESSION)
  • profil.php : lecture/écriture du profil utilisateur
  • creer_cv.php : formulaire de construction du CV
  • generate_cv.php : traitement + export PDF via TCPDF
  • offres.php, annonces.php, postuler.php : CRUD offres + interaction utilisateurs
  • candidatures.php : affichage des candidatures
  • competences.php, suivi_competences.php : système de tagging/notation
  • messages.php : messagerie interne

1.3 Système d'authentification

  • Stockage des mots de passe en SHA-256 ou BCRYPT (via password_hash)
  • Sessions initiées avec session_start()
  • Vérification de rôle ($_SESSION['role']) pour accéder aux sections (admin, recruteur, candidat)

1.4 Exemple de code PHP : Authentification

session_start();
include 'config.php';
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
  $email = $_POST['email'];
  $password = $_POST['password'];
  $stmt = $pdo->prepare("SELECT * FROM utilisateurs WHERE email = ?");
  $stmt->execute([$email]);
  $user = $stmt->fetch();
  if ($user && password_verify($password, $user['mot_de_passe'])) {
    $_SESSION['user_id'] = $user['id'];
    $_SESSION['role'] = $user['role'];
    header('Location: dashboard.php');
  } else {
    $error = "Identifiants invalides";
  }
}

II. Base de données relationnelle alternance_it

2.1 Principales entités

  • utilisateurs : identifiants, rôles, mot de passe hashé, email
  • profil_utilisateur : données enrichies : école, ville, objectif, lien LinkedIn, portfolio
  • annonces : titre, contenu, recruteur_id, date publication
  • candidatures : statut, date, FK vers utilisateurs et annonces
  • competences : table référentielle, catégorie incluse
  • utilisateurs_competences : clé composite, niveau 1–5
  • annonces_competences : compétences exigées par offre
  • experiences_professionnelles : données chronologiques pour CV
  • messages : échanges internes entre utilisateurs

2.2 Requêtes SQL utiles

Création de la table utilisateurs

CREATE TABLE utilisateurs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nom VARCHAR(50),
  email VARCHAR(100) UNIQUE NOT NULL,
  mot_de_passe VARCHAR(255) NOT NULL,
  role ENUM('candidat', 'recruteur', 'admin') DEFAULT 'candidat',
  date_creation DATETIME DEFAULT CURRENT_TIMESTAMP
);

Jointure compétences ↔ offres

SELECT DISTINCT a.*
FROM annonces a
JOIN annonces_competences ac ON a.id = ac.annonce_id
JOIN utilisateurs_competences uc ON ac.competence_id = uc.competence_id
WHERE uc.utilisateur_id = :id_utilisateur;

Historique de candidatures

SELECT c.*, a.titre
FROM candidatures c
JOIN annonces a ON c.annonce_id = a.id
WHERE c.utilisateur_id = ?;

III. Serveur Web embarqué – ESP32 WROVER

3.1 Stack utilisée

  • Microcontrôleur : ESP32 WROVER (WiFi/BLE, 16 Mo flash)
  • IDE : Arduino IDE + ESP32 board support
  • Librairies principales :
    • ESPAsyncWebServer : serveur HTTP non-bloquant
    • SPIFFS / SD.h : gestion de fichiers via carte microSD
    • Hash.h : SHA-256

3.2 Code Arduino – Setup du serveur

#include <WiFi.h>
#include <ESPAsyncWebServer.h>
#include <SD.h>

AsyncWebServer server(80);

void setup() {
  Serial.begin(115200);
  WiFi.begin("SSID", "PASSWORD");
  while (WiFi.status() != WL_CONNECTED) delay(1000);
  SD.begin();
  server.on("/", HTTP_GET, [](AsyncWebServerRequest *request){
    request->send(SD, "/index.html", "text/html");
  });
  server.begin();
}

IV. Sécurité et bonnes pratiques

  • Sanitation côté PHP avec htmlspecialchars, filter_var pour les inputs
  • Prepared statements (PDO ou mysqli avec bind_param) pour éviter les injections SQL
  • Authentification sur ESP32 avec cookie unique + hash SHA-256, non exposé dans les URL
  • Aucune donnée sensible stockée sur ESP32 en clair (mot de passe ou identifiants)
  • Séparation des rôles : accès conditionné aux pages par niveau (admin, user, recruiter)

V. Conclusion technique

Ce projet démontre la faisabilité d’un environnement de gestion complet :

  • 100% PHP/MySQL sans framework (maîtrise native)
  • Intégration embarquée locale via ESP32 avec API REST
  • Couverture complète de la logique métier : utilisateur ↔ offre ↔ compétence

Il constitue une base solide pour un projet de déploiement pédagogique, d’ateliers web embarqué, ou de MVP orienté alternance/emploi. Des améliorations futures incluraient :

  • Passage à HTTPS via proxy externe (NGINX)
  • Optimisation de la base (normalisation 3FN, triggers)
  • Portage vers React ou Vue.js pour le frontend
  • Gestion dynamique des fichiers ESP32 via SPIFFS OTA
